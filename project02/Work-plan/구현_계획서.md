# "오늘 뭐 먹지?" 결정 장애를 위한 맛집 추천 사이트 구현 계획
사용자의 결정 장애를 돕기 위해 직관적인 마우스 인터랙션을 통하여 시각적인 탐색 경험을 제공하는 맛집 추천 서비스(웹 애플리케이션)입니다.

## User Review Required

> [!IMPORTANT]
> 프로젝트 설정 및 구체적인 디자인 컨셉에 대해 다음 내용을 확인해 주세요.

1. **개발 스택 (프레임워크) 및 애니메이션**
   - 고품질의 동적 웹사이트 구축을 위해 **Next.js** 또는 **React + Vite** 사용이 권장됩니다. 이 중 선호하는 것이 있으신가요? (지정이 없다면 가볍고 빠른 React + Vite 스택으로 먼저 진행하겠습니다.)
   - iheartcomix.com 처럼 역동적이고 동적인 인터랙션(마우스 트래킹, 자유배치, 회전 등) 구현을 위해 바닐라 CSS 외에 **Framer Motion이나 GSAP**와 같은 애니메이션 라이브러리를 추가로 사용해 구성하겠습니다.

2. **맛집 데이터 및 위치 권한**
   - 개발 초기 단계에서는 UI/UX 구현에 우선 집중하기 위해 **임의의 테스트 데이터(Mock Data)** 10여 개 이상을 카테고리 및 무드별로 만들어 진행하고자 합니다.
   - "가장 가까운" 식당 정렬 기능을 위해 브라우저의 사용자 정보(Geolocation)를 활용할 예정입니다. 메인페이지 혹은 상세페이지 진입 시 위치 권한을 브라우저에 요청할 계획입니다.

## Proposed Changes

### 1. 메인 페이지 (Main Page)
- **핵심 컨셉**: iheartcomix.com의 캔버스 기반의 인터랙티브하고 트렌디한 레이아웃 차용. 디자인적으로 심미성이 아주 돋보이도록 구성합니다.
- **UI 구성**:
  - 요청하신 대로 화면 가운데 버튼 영역 완전 제거.
  - **타이틀/서브타이틀**: 화면을 열었을 때 명확히 인식되도록 "오늘 뭐 먹지?", "마우스를 움직여 탐색해보세요" 타이포그래피 정중앙 배치.
- **인터랙션 요소**:
  - **이미지 영역**: [한식, 양식, 일식, 분식, 디저트] 등 주요 음식 카테고리를 상징하는 고해상도 음식 사진 이미지가 화면 곳곳에 랜덤하게 떠다니거나 마우스를 쫓아 반응하여 나타나도록 처리.
  - **텍스트 영역**: [매콤, 느끼, 가성비, 플렉스, 혼밥, 로컬맛집] 등의 텍스트가 시각적으로 흥미롭게 화면을 채우고, 마우스 오버 시 역동적인 피드백(색상 반전, 폰트 두께 변화 등) 제공.
- **라우팅 동작**: 각각의 이미지나 텍스트 요소를 클릭 시 해당 속성을 파라미터로 하여 상세 페이지(`/detail?tag=...`)로 매끄럽게 넘어갑니다.

### 2. 상세 페이지 (Detail Page)
- **핵심 컨셉**: 식신(siksinhot.com) 스타일로 명확하고 신뢰감 있는 맛집 리스트업. 최소 10개 이상의 퀄리티 높은 항목 제공.
- **UI/UX 요건**:
  - 선택한 분류(예: "매콤한 맛집 10선" 혹은 "일식 맛집")에 맞추어 상단에 결과 타이틀 크게 출력.
  - **정렬 옵션 토글**: 리스트 상단에 '가장 가까운 순(Distance)' / '가장 인기있는 순(Rating)' 탭 버튼 구성.
  - **리스트 카드 디자인**:
    - 직관적이고 군침도는 음식(식당) 썸네일
    - 식당 이름 및 별점 (예: ⭐️ 4.8 / 5.0)
    - 상세 카테고리 태그 모음 (예: `#양식` `#로컬맛집`)
    - 위치 정보 데이터 연동으로 현재 위치 기준 거리 (예: `도보 12분`, `0.8km`)
- **디자인 미학**: 바닐라 CSS 기반으로 세련된 모던 디자인, 부드러운 호버 애니메이션, 모바일/데스크톱 대응 반응형 그리드 및 모던 폰트를 적용해 고급스러운 느낌 연출.

## Verification Plan
1. **상호작용 품질 검사**: 메인 진입 시 사진과 키워드들이 제대로 배치되고 충돌 없이 마우스 이벤트에 반응하는지 검증.
2. **상세 페이지 연결 검사**: 클릭한 조건(이미지 분류, 텍스트 무드)이 정확하게 상세 목록으로 전달 및 렌더링 되는지 확인.
3. **데이터 필터 정렬 테스트**: 최소 10개의 리스트가 거리순, 평점순에 따라 오류 없이 재정렬되는지 테스트 수행.
